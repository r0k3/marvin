# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | Yes                |

## Reporting a Vulnerability

If you discover a security vulnerability in Marvin, please report it responsibly.

**Do not open a public issue.**

Instead, use [GitHub Security Advisories](https://github.com/r0k3/marvin/security/advisories/new) to privately report the vulnerability. This ensures the issue can be assessed and patched before public disclosure.

### What to include

- A description of the vulnerability and its potential impact
- Steps to reproduce the issue
- Any relevant logs, screenshots, or proof-of-concept code

### Response timeline

- **Acknowledgement:** Within 7 days of your report
- **Status update:** Within 14 days, with an assessment and remediation plan
- **Fix release:** As soon as a patch is ready, coordinated with the reporter

Thank you for helping keep Marvin and its users safe.
